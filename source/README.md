## How to Update

### zh-CN_R466.csv

ACE (Actions, Conditions, Expressions) translation file.

### scripts/ (TypeScript definitions & templates)

TypeScript definitions for runtime scripting API. Generated by Construct 3 Editor.

#### Initial Setup (New Project)

1. Create a new Construct 3 project
2. Save project as **folder** format (not .c3p)
3. Right-click **Scripts** folder in Project Bar
4. Select `TypeScript › Set up TypeScript for external editor`
5. Construct generates:
   - `scripts/ts-defs/` - TypeScript definitions
   - `scripts/tsconfig.json` - TypeScript config
   - `scripts/main.ts` - Entry point template
   - `scripts/importsForEvents.ts` - Event imports template

#### Update Existing Definitions

When Construct 3 version updates:

1. Open project in latest Construct 3
2. Right-click **Scripts** folder in Project Bar
3. Select `TypeScript › Update TypeScript definitions`
4. Copy updated `scripts/ts-defs/` folder

### ts-defs Structure

```

├── ts-defs/
│   ├── runtime/                # Runtime API interfaces (38 files)
│   │   ├── IRuntime.d.ts       # Main runtime entry point

│   │   ├── IWorldInstance.d.ts # World instance (position, size, angle)
│   │   ├── ILayout.d.ts        # Layout management
│   │   ├── ILayer.d.ts         # Layer properties
│   │   ├── IObjectClass.d.ts   # Object type access
│   │   ├── IBehaviorInstance.d.ts
│   │   ├── IAnimation.d.ts
│   │   ├── IRenderer.d.ts
│   │   └── ...
│   ├── objects.d.ts            # Project object types (auto-generated stub)
│   ├── globalVars.d.ts         # Global variables (auto-generated stub)
│   ├── layers.d.ts             # Layer types (auto-generated stub)
│   ├── layouts.d.ts            # Layout types (auto-generated stub)
│   ├── instanceTypes.d.ts      # Instance types (auto-generated stub)
│   ├── eases.d.ts              # Tween eases (auto-generated stub)
│   └── localVars.d.ts          # Local variables (auto-generated stub)
├── main.ts                     # Script entry point template
├── importsForEvents.ts         # Functions for event sheet scripts
└── tsconfig.json               # TypeScript compiler config
```

## Notes

- `ts-defs/runtime/` contains the authoritative runtime API definitions
- Project-specific files (`objects.d.ts`, `layers.d.ts`, etc.) are empty stubs for TypeScript compilation
- Update these files when Construct 3 releases major API changes
- See [Using an external editor](https://www.construct.net/en/make-games/manuals/construct-3/scripting/guides/using-external-editor) for official documentation
